# Support

## Before Opening an Issue

Please take a moment to check the docs and existing issues before opening a new one. Most common problems are covered in [`Docs/TROUBLESHOOTING.md`](Docs/TROUBLESHOOTING.md), and a quick search of existing issues often turns up a prior discussion.

---

## What Belongs in an Issue

**Bug reports** — Something that used to work and now doesn't, or something that never worked but should. Please use the bug report template. Include:
- A description of the problem and what you expected to happen
- Your environment (Xcode version, tvOS version, Apple TV model, WebRTC revision from `ThirdParty/WebRTC/webrtc-version.json`)
- Console logs with relevant `[Auth]`, `[Stream]`, `[WebRTC]`, or `[AudioStats]` lines

**Feature requests** — Something the project does not currently do that you believe it should. Use the feature request template. Check [`Docs/FEATURE_INVENTORY.md`](Docs/FEATURE_INVENTORY.md) and [`Docs/FUTURE_WORK.md`](Docs/FUTURE_WORK.md) first to see if it is already tracked.

---

## What Does NOT Belong in an Issue

- Questions about how to use Xbox Game Pass or xCloud — Microsoft's own support handles those
- Questions about setting up an Xbox Game Pass Ultimate subscription
- Requests for features that require Microsoft's private APIs or signing keys
- General Swift or tvOS development help unrelated to this project

---

## Security Issues

**Do not open a public issue for security vulnerabilities.** Use GitHub's private vulnerability disclosure instead: see [`SECURITY.md`](SECURITY.md) for the reporting process.

---

## Questions and Discussion

For open-ended questions, contribution ideas, or general discussion about the project, [GitHub Discussions](https://github.com/nafields/cloudx/discussions) is the right place. Issues are for tracked bugs and concrete feature requests — Discussions is for everything else.

---
